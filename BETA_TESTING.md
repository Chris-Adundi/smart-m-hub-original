# Smart M Hub Beta Testing Guide

This guide is for invited testers using the public beta links.

## Test Links

The deployment owner should provide:

- School portal link
- Super admin dashboard link, for platform-owner testers only
- Support or feedback channel

## How to Register a School

1. Open the school portal link.
2. Select `Register School`.
3. Complete the school registration form with realistic test data.
4. Submit the registration.
5. Wait for platform approval before trying to use the school account.

## How to Log In

After approval:

1. Open the school portal link.
2. Select `Access Portal`.
3. Enter the school code, email, and password.
4. Use the role-specific dashboard assigned to your account.

Student or parent accounts may use either:

- Email and password with the unique student access code, or
- Email and password with the admission number.

## What to Test

School admin:

- School profile and branding
- Pending user approvals
- Student approvals
- Fee receipt approvals
- Attendance/result/announcement approvals
- Support notices and platform announcements

Secretary:

- Student admission
- Class assignment
- Guardian and document records
- Student access code visibility

Teacher:

- Assigned class visibility
- Attendance submission
- Exam and assessment report upload
- Student history and approved records

Finance:

- Student fee records
- Payment entry
- Receipt upload
- Fee status colors and admin approval workflow
- Fee structures

Student or parent:

- Announcements
- Fee status
- Approved payment receipts
- Approved exam or assessment downloads

Super admin:

- School approvals
- Analytics
- Payments
- Support tickets
- System health
- Diagnostics center

## Feedback Format

Send feedback with:

- Device and browser
- Role used
- School code
- Page or feature tested
- What happened
- What you expected
- Screenshot or screen recording if possible
- Time and date of the issue

## Beta Data Notice

Use test data only. Do not enter real student medical information, private national IDs, real payment references, or sensitive school records during beta testing unless the deployment owner confirms that the beta environment is approved for that data.

## Known Beta Rules

- New schools require platform approval.
- New school users require school admin approval.
- Student and parent accounts should only see their own linked student records.
- Receipts, attendance, exams, and assessments appear in the student portal only after approval.
